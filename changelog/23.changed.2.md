- ⚠️ Switched the wire protocol to a snapshot + per-worker lifecycle model. A new client receives a single `snapshot` event with `protocol`, runtime versions, and currently-registered workers, followed by live `worker_added` / `worker_status` / `worker_removed` / `frame` / `bus_event` deltas. Per-worker topology now travels inside `worker_added` (no more top-level `pipeline` message), and the recording file format follows the same wire encoding so `WhiskerFile` captures replay through the same UI as a live session.
